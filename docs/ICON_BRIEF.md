# 轻舟 App 图标 — 设计 Brief

> 这一份发给设计师。包含必交付清单 + 风格指引 + 反面教材。

---

## 1. 项目背景

**App 名**: 轻舟 / Qingzhou
**寓意**: 出自李白「轻舟已过万重山」—— 暗喻轻松跨越障碍,不必把"障碍"画出来
**用途**: 一个跨平台 iOS / macOS 工具类 App,运行在系统后台,帮用户管理本机网络配置
**目标用户**: 25–40 岁、技术 / 创意行业、对设计敏感的中产专业人士
**目标市场**: 国际 App Store(主要为美 / 港 / 日 / 新 / 加),用户以海外华人为主

## 2. 必交付清单

| # | 用途 | 尺寸 | 格式 | 备注 |
|---|---|---|---|---|
| 1 | App Store 列表图标(主图) | **1024 × 1024 px** | PNG (无透明、无圆角、无 alpha 通道) | Apple 自动加圆角 |

## 3. 视觉风格

### 3.1 元素

主体: **一艘极简风格的小舟**(剪影 / 几何抽象 / 单线条均可),处于一个清晰可辨的环境里(水面 / 山影 / 月光 / 留白)。

**不要包含**:
- ❌ 网络 / 地球仪 / 信号符号(暗示 VPN)
- ❌ 锁 / 盾牌 / 防火墙(暗示安全 / 翻墙工具)
- ❌ 大写英文字母 Q(冗余)
- ❌ 中文 "轻舟" 二字(图标已经有名字,不要重复)
- ❌ 任何写实复杂插画(在 60×60 缩到 Home Screen 上认不出来)

### 3.2 配色

候选方向(选一个,不要混):

**方向 A — 静海蓝**
- 主色: `#3A6FBE`(深海蓝)
- 副色: `#E8F2FF`(雪雾白)
- 点缀: 月光银 / 极少金线
- 调性: 清晨的湖面,稳重内敛

**方向 B — 落墨**
- 主色: `#2C2E33`(墨黑)
- 副色: `#F2EFE6`(宣纸白)
- 点缀: 朱砂红色一抹(中国画落款感)
- 调性: 文人画,极简,适合 macOS 状态栏

**方向 C — 黄昏渡**
- 主色: `#D67F47`(暖橘)
- 副色: `#1F2D3D`(深夜蓝)
- 点缀: 暖白
- 调性: 黄昏小舟,情绪浓郁

**避坑**:
- ❌ 不要纯红色为主(中国市场某些政治场合敏感)
- ❌ 不要使用三色及以上(图标小尺寸辨识度下降)
- ❌ 不要用 iOS 系统色(蓝色 #007AFF / 紫色 #5856D6 之类),容易跟系统 app 混

### 3.3 风格参考

类似定位的优秀 App 图标可以参考(请仅借鉴气质,不要抄):

- [Things 3](https://culturedcode.com/things/)(Cultured Code) — 极简、单一物件、留白
- [Bear](https://bear.app/)(Shiny Frog) — 简洁、品牌色记忆点强
- [Reeder](https://reederapp.com/) — 经典几何
- 中国画里的小舟构图: 看 [马远《寒江独钓》](https://zh.wikipedia.org/wiki/%E5%AF%92%E6%B1%9F%E7%8D%A8%E9%92%93%E5%9B%BE)

## 4. iOS Human Interface Guidelines — 必须遵守

1. **铆定方形,Apple 加圆角**: 主图必须是方形,不要自己加圆角,iOS 会按它的 squircle 规则圆角
2. **不要描边**: iOS 14 起 Apple HIG 明确建议不要给图标加 1px 描边或边框
3. **足够的对比**: 在浅色 / 深色 Home Screen 上都要清晰可读 —— 别用浅灰背景配白图形
4. **细节适应缩小**: 至少要在 87×87(iOS 大图标)和 60×60(iOS 标准)尺寸下做缩略图测试,
   主体在 60×60 下仍可辨认
5. **不带 alpha 通道**: PNG 导出时 export 选 "No transparency"。带 alpha 提交会被 Apple 拒
6. **sRGB 色彩空间**: 不要 P3 / Adobe RGB,导出前确认颜色配置

---

## 5. 反面教材(常见拒绝点)

- **山寨痕迹**: 直接抄 Outline / Shadowrocket / Surge 的图标会被 Apple 标 4.1(版权)
- **iOS 系统图标元素**: 不要用 Apple 内置的 SF Symbol 直接放进图标
- **截图风格的"应用界面预览图"**: 有些设计师会把 app UI 缩小放进图标。不要,违反 HIG
- **极小文字**: 写小字在 60×60 下完全模糊,基本没用还显得 cheap
- **3D 拟物质感**: 跟 iOS 17+ 系统调性不一致,会显得过时

---

## 6. 工作流程建议

1. **草图阶段**(0.5–1 天): 设计师出 3–5 个不同方向(A/B/C 配色 + 不同构图)的低保真草图
2. **方向选定**(0.5 天): 你从 3–5 个里挑 1–2 个,反馈
3. **精修**(1–2 天): 出高保真 1024×1024 主图 + 在 87×87 和 60×60 缩略效果图
4. **微调 + 终稿**(0.5–1 天)
5. **导出全套尺寸**(自动化,0.5 天)

总时长: **3–5 个工作日**。

## 8. 验收 checklist

收到稿后逐项确认:

- [ ] 1024×1024 PNG 主图,sRGB,无 alpha 通道
- [ ] 在 Mac 上拖到桌面看,缩到 64×64 后主体仍可辨认
- [ ] 在 iOS 设备上用 [Icon Preview](https://apps.apple.com/app/icon-preview/id1500200831) app 预览
   或直接 build 进 Xcode 装真机看 Home Screen 实际效果
- [ ] 暗色 / 浅色 Home Screen 背景下对比都没问题
- [ ] 不含任何文字 / 网络元素 / 锁元素
- [ ] 源文件(Figma 链接 / .sketch / .ai)留着,以后改字体颜色不用重画
