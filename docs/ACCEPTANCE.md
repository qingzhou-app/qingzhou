## 1. #11B libXray 能力接入 · 四项（已合 main @ 1106005）

> 背景：合并后 526 单测全绿 + iOS/macOS 双平台编译通过，但四项功能都没在真机上人工验收过。

### 1.1 经代理延迟

- **前置条件**：VPN 已开启且节点可用；设置页诊断区可见扩展内存 footprint
- **操作步骤**：
  1. 节点页长按某节点 → 「测经代理延迟」
  2. 工具栏「⋯」→ 批量测经代理延迟，观察进度
  3. 测速过程中去 设置 → 诊断区 看扩展内存 footprint
  4. 关闭 VPN，回到节点页确认入口状态；再跑一次原直连测速
- **预期结果**：
  - 长按单测后该节点出现**第二个延迟 chip**（经代理延迟，与直连延迟并列）
  - 批量测速有进度反馈，全部节点跑完
  - **扩展内存 footprint 不逼近 40MB**（50MB 硬上限，超了系统杀进程=断流）
  - 关 VPN 时「经代理延迟」入口禁用；原直连测速行为不变
- **验收状态**：
  - [ ] 通过（日期/设备：＿＿＿）

### 1.2 配置预检

- **前置条件**：准备一个**坏配置节点**（如故意改错端口/密码的节点）+ 一个好节点
- **操作步骤**：
  1. 连着好节点（VPN 开启）时，切换到坏配置节点
  2. 观察提示与网络连通性（切换时开着一个持续 ping / 视频流更直观）
  3. 关 VPN → 直接选坏节点冷启动 VPN
- **预期结果**：
  - 热切换到坏节点：提示「已保持当前节点连接不变…」且**不断网**（当前连接保持）
  - 冷启动坏节点：alert 显示 **xray 可读错误信息**（`fetchLastDisconnectError` 链路），
    不是干巴巴的「连接失败」
- **验收状态**：
  - [ ] 通过（日期/设备：＿＿＿）

### 1.3 代理 / 直连流量拆分（QueryStats）

- **前置条件**：规则代理模式；VPN 开启并产生一些混合流量（访问国内站 + 国外站）
- **操作步骤**：
  1. 规则模式下开 VPN，浏览若干国内 + 国外网站
  2. 回首页看流量卡
  3. 关 VPN，等几秒再看流量卡
  4. 顺带观察：VPN 每次启动是否正常（metrics inbound 拿不到端口时应静默不开统计，
     不能影响 VPN 启动）
- **预期结果**：
  - 首页流量卡出现「代理 / 直连」行 + 各自占比，数字随访问增长
  - 关 VPN 几秒后「代理 / 直连」行消失
  - VPN 启动稳定，无因统计端口问题导致的启动失败
- **验收状态**：
  - [ ] 通过（日期/设备：＿＿＿）

### 1.4 节点导出（反向导出分享链接）

- **前置条件**：节点列表里有多种协议的节点（trojan / vmess / vless / ss / hy2 尽量都有）
- **操作步骤**：
  1. 单节点 → 「复制分享链接」→ 粘贴回「添加节点」
  2. 「导出全部节点链接」→ 全部粘贴回「添加节点」
  3. （加分项）把导出的链接粘到其他客户端（v2rayN 等）导入一次；
     再从其他客户端导出链接粘回轻舟
- **预期结果**：
  - 粘回后**全量解析成功**，节点参数（协议/主机/端口/加密/传输层参数）与原节点一致
  - 与其他客户端互导无字段丢失（加分项，不阻塞验收）
- **验收状态**：
  - [ ] 通过（日期/设备：＿＿＿）

## 4. iCloud 恢复弹窗降噪（commit 54aae48）

> 修复内容：云端与本地内容哈希一致时**静默采认**不弹窗；点「暂不恢复」按云端 revision
> 持久化记忆；只有云端**真有用户内容变化**才再次弹恢复确认。

- **前置条件**：iPhone + Mac 同一 iCloud 账号，两端都装有包含该修复的构建，iCloud 保险柜已启用
- **操作步骤**：
  1. 制造一次恢复确认弹窗（如 Mac 改配置后 iPhone 冷启动），iPhone 弹出恢复确认时点**「暂不恢复」**
  2. 杀掉 iPhone 主 App，重新打开
  3. 在 Mac 上**改订阅**（真实内容变化），等同步后杀掉 iPhone App 重开
  4. 在 iPhone 上执行一次恢复；然后在 Mac 上**只改主题再改回去**（内容净变化为零），
     杀掉 iPhone App 重开
- **预期结果**：
  - 步骤 2：**不应再弹**恢复确认（拒绝已按 revision 持久化）
  - 步骤 3：**应弹**恢复确认（云端有真实用户内容变化）
  - 步骤 4：**不应弹**（内容哈希一致，静默采认）
- **验收状态**：
  - [ ] 通过（日期/设备：＿＿＿）
